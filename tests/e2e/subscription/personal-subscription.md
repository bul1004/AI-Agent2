### 個人モードでのサブスクリプション (`personal-subscription`)

- 組織に所属していない個人ユーザーがサブスクリプションを契約できることを確認

#### テストケース

| テスト名                                       | 確認事項                                                         |
| ---------------------------------------------- | ---------------------------------------------------------------- |
| `個人モードでプラン詳細が正しく表示される`     | 「/月」表記、「個人事業主向けプラン」、「個人プランを始める」ボタン |
| `個人モードでStripeチェックアウトにリダイレクトされる` | 「個人プランを始める」クリックでStripeへリダイレクト             |

#### 手順（プラン詳細表示確認）

1. 新規テストユーザーを作成してログイン状態にする（組織なし = 個人モード）
2. サイドバーのプロフィールボタンをクリック
3. 「プランをアップグレード」メニューをクリック
4. モーダルが開き、以下が表示されることを確認:
   - 「/月」の表記（「/シート/月」ではない）
   - 「個人事業主向けプラン」のテキスト
   - 「個人プランを始める」ボタン
   - プラン料金（¥9,800）
   - 「30日間返金保証」のテキスト

#### 手順（Stripeチェックアウトリダイレクト）

1. 新規テストユーザーを作成してログイン状態にする（組織なし = 個人モード）
2. サブスクリプションモーダルを開く
3. 「個人プランを始める」ボタンをクリック
4. `checkout.stripe.com` にリダイレクトされることを確認

#### 前提条件

- なし（テストごとに新規ユーザーを作成）
- Stripe テスト環境が設定済み
- `STRIPE_SECRET_KEY`, `STRIPE_PRICE_ID_BUSINESS` が環境変数に設定済み

#### 使用するページオブジェクト

| クラス        | 用途                                   |
| ------------- | -------------------------------------- |
| `BillingPage` | サブスクリプションモーダルの操作・検証 |

#### 使用するヘルパー関数

| 関数名            | 用途                                       |
| ----------------- | ------------------------------------------ |
| `createTestUser`  | テストユーザーを作成してログイン状態にする |
| `cleanupTestUser` | テストユーザーをクリーンアップする         |

#### 備考

- 各テスト後に `cleanupTestUser` でテストユーザーをクリーンアップ
- 個人モードでは `userId` を課金対象IDとして使用
- 個人ユーザーは組織を作成せずに直接課金可能
