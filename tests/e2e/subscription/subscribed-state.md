### 契約済み状態でのサブスクリプション表示 (`subscribed-state`)

- 契約済みユーザーに対する表示が正しいことを確認

#### テストケース

| テスト名                               | 確認事項                                   |
| -------------------------------------- | ------------------------------------------ |
| `契約済みの場合「契約中」ボタンが表示される` | 契約済み時に「契約中」ボタンが表示される     |
| `契約済みでもプラン詳細は表示される`   | 契約済みでもプラン情報は閲覧可能             |

#### 手順（契約中ボタン確認）

1. テスト前にDBにサブスクリプション（plan=business, status=active）を設定
2. シードデータのオーナーユーザー（e2e-owner@example.com）でログイン
3. 「E2E Test Team」を選択してチームモードに切り替え
4. プロフィールボタンをクリック
5. 「プランをアップグレード」メニューをクリック
6. モーダルが開き、「契約中」ボタンが表示されることを確認
7. ボタンが無効化されていることを確認

#### 前提条件

- シードデータが投入済み（`supabase db reset`）
- テスト実行時にサブスクリプションレコードを自動作成

#### 使用するシードデータ

| ロール | メール                | パスワード          | 組織          |
| ------ | --------------------- | ------------------- | ------------- |
| owner  | e2e-owner@example.com | E2eTestPassword123! | E2E Test Team |

#### 使用するヘルパー関数

| 関数名                  | 用途                                |
| ----------------------- | ----------------------------------- |
| `loginAsUser`           | シードデータのユーザーでログイン    |
| `switchToOrganization`  | チームモードに切り替え              |
| `setTestSubscription`   | テスト用サブスクリプションをDBに設定 |
| `cleanupTestSubscription` | テスト用サブスクリプションを削除   |

#### 備考

- テスト前後でサブスクリプションレコードを作成・削除
- 実際のStripe連携ではなくDBを直接操作
