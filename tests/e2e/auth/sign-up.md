### メールサインアップ (`sign-up`)

- ランディングページからのサインアップフロー、メールでの新規ユーザー登録が正しく動作することを確認

#### テストケース

| テスト名                                                    | 確認事項                                                 |
| ----------------------------------------------------------- | -------------------------------------------------------- |
| `ランディングページのヘッダーからサインアップページへ遷移できる` | ヘッダーの「無料で始める」ボタンからサインアップページへ遷移 |
| `ランディングページのモーダルからサインアップページへ遷移できる` | チャット入力フォーカス→モーダル→「メールで登録」で遷移   |
| `新規ユーザーがメールでサインアップできる`                   | メールアドレスとパスワードで新規登録→/chat へリダイレクト |
| `パスワードが8文字未満の場合エラーが表示される`              | 短いパスワードでサインアップページに留まる               |

#### 手順（ヘッダーからサインアップ）

1. ランディングページ（`/`）に移動
2. ヘッダーの「無料で始める」ボタンをクリック
3. `/signup` へ遷移することを確認
4. サインアップフォームが表示されることを確認

#### 手順（モーダルからサインアップ）

1. ランディングページ（`/`）に移動
2. チャット入力フィールドをフォーカス
3. 認証モーダルが表示されることを確認
4. 「メールで登録」ボタンをクリック
5. `/signup` へ遷移することを確認
6. サインアップフォームが表示されることを確認

#### 手順（メールサインアップ - ハッピーパス）

1. サインアップページ（`/signup`）に移動
2. 名前を入力（例: `テスト ユーザー`）
3. メールアドレスを入力（ユニークなテストメール）
4. パスワードを入力（8文字以上）
5. サインアップボタンをクリック
6. `/chat` へリダイレクトされることを確認

#### 手順（パスワードバリデーション）

1. サインアップページ（`/signup`）に移動
2. 名前を入力
3. メールアドレスを入力
4. 8文字未満のパスワード（例: `short`）を入力
5. サインアップボタンをクリック
6. `/signup` ページに留まることを確認（リダイレクトされない）

#### 前提条件

- なし（テストごとに新規ユーザーを作成）

#### 使用するページオブジェクト

| クラス        | 用途                           |
| ------------- | ------------------------------ |
| `LandingPage` | ランディングページの操作       |
| `SignupPage`  | サインアップページの操作・検証 |

#### 備考

- 各テスト後に `cleanupTestUser` でテストユーザーをクリーンアップ
- fixtures.ts の `generateTestEmail` でユニークなメールアドレスを生成
- タイムスタンプ + ランダム文字列でメール重複を回避
